# Платформа для курсов ORM

Учебный проект платформы для управления онлайн-курсами, демонстрирующий возможности JPA/Hibernate с использованием Spring Boot.

## Функциональные возможности

- **Управление пользователями**: Студенты, Преподаватели, Администраторы.
- **Управление курсами**: Категории, Курсы, Модули, Уроки.
- **Учебный процесс**: Записи на курсы, Задания, Сдачи заданий, Тесты.
- **База данных**: PostgreSQL со сложными связями (Один-к-одному, Один-ко-многим, Многие-ко-многим).

## Технологический стек

- **Java**: 17
- **Фреймворк**: Spring Boot 3.2.1
- **ORM**: Hibernate / Spring Data JPA
- **База данных**: PostgreSQL
- **Тестирование**: JUnit 5, Testcontainers

## Предварительные требования

- Docker (должен быть запущен для тестов и локального запуска)
- Java 17+

## Как запустить

### 1. Запуск базы данных (Вручную)

Если вы хотите запустить приложение вручную:

```bash
docker-compose up -d
```

Это запустит PostgreSQL на порту `5432`.

### 2. Запуск приложения

Вы можете запустить приложение, используя вашу IDE или Maven wrapper (если установлен).

```bash
# Если у вас установлен mvn
mvn spring-boot:run
```

## Тестирование

Проект настроен с использованием **Testcontainers**. Это означает, что вам **не нужно** вручную запускать базу данных для прогона тестов. Тесты автоматически поднимут Docker-контейнер с PostgreSQL.

```bash
# Запуск интеграционных тестов
mvn test
```

### Что тестируется?
`OrmCourseApplicationTests` демонстрирует:
1.  Запуск контекста с контейнеризированной БД Postgres.
2.  Создание пользователей (Преподаватель, Студент).
3.  Создание курса со связями.
4.  Запись студента на курс.
5.  Проверку состояния базы данных.

## Структура проекта

- `model`: Эндтити JPA (User, Course, Module и т.д.)
- `repository`: Репозитории Spring Data
- `service`: Бизнес-логика
- `config`: Конфигурация приложения

## Схема базы данных (Ключевые сущности)

- **User**: Главное действующее лицо.
- **Course**: Центральная сущность, связанная с преподавателем и категорией.
- **Enrollment**: Связь между пользователем и курсом.
- **Module/Lesson**: Иерархия контента.
